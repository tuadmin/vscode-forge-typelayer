# Forge TypeLayer v1.0.0

**The Pragmatic Switch for the Real-World Engineer.**

A VS Code extension designed specifically for developers who value absolute
control, but realize that pragmatic reality often demands speed.

Some days you wake up as a purist: writing abstract classes, interfaces, and
maintaining an iron-clad TypeScript architecture. Other days, you have ruthless
deadlines and just need to sling plain JavaScript at the speed of
light—sprinkling JSDoc for quick checks.

**Forge TypeLayer is the ultimate zero-config bridge.** It allows you to use a
strict TypeScript layer as your private "source of truth", while seamlessly
exposing plain, unencumbered JavaScript directly to compilers, web browsers, or
frameworks—keeping your final contracts pure and agnostic.

> [!NOTE]
> **Scope Clarification:** Forge TypeLayer is designed specifically for **Mixed
> ecosystems** (where your main runtime expects pure JavaScript, but you want
> TypeScript as a source-of-truth companion). If you are building a 100% pure
> TypeScript ecosystem (where the bundler or runtime handles TS transparently),
> you likely won't need this extension.

---

## 🛠 Features for the Trenches

### 1. Instant Type Forging

Why battle Webpack or heavy Bundlers just to prototype? Your workspace types are
strictly validated, but what goes out to the world is raw, pristine JavaScript.
**No build step blockades.**

### 2. Intelligent Auto-Watermarking

Ever clicked "Go to definition" in VS Code, ended up in the `.js` or `.d.ts`
output file, edited it manually, and then lost all your work on the next
compile? Forge TypeLayer automatically injects a
`DO NOT EDIT DIRECTLY – Autogenerated` watermark lock on all exported artifacts
so that your team knows exactly where the _Source of Truth_ really lives.

### 3. Smart SSD Short-Circuiting

If you create a file just to drop a `// TODO` or some comments, Forge TypeLayer
is smart enough to abort TypeScript execution instantly. It won't wake up
background TS servers or burn your SSD’s lifespan doing pointless I/O for empty
files. It only works when real code is present.

### 4. Zero Config (It just works)

Under the hood, if you have `bunx tsc` or `npx tsc`, we use it. If you don't, we
use an ultra-fast internal memory API fallback to reconstruct the logic
perfectly. You never have to configure complex `tsconfig.json` mappings for
basic projects again.

## ⚙️ How it works

Create your source file (e.g., `foo.mts` or `foo.f.ts`) as the primary source of
truth. When you save (or trigger the command), Forge TypeLayer automatically
constructs cleanly matched artifacts:

**For Modern ESM (`foo.mts`):**

1. `foo.mjs`: The raw ES Module logic layer, stripped of types.
2. `foo.d.mts`: The type layer for strictly exporting ESM definitions.

**For The Forge Lock (`foo.f.ts`):**

1. `foo.js`: The raw standard logic layer.
2. `foo.d.ts`: Standard JS type definitions. _(Notice how the `.f.` lock is
   cleanly stripped from the output!)_

This enables consumers to directly import pure JavaScript while VS Code
maintains a flawless intellisense experience using the adjacent definition
files.
